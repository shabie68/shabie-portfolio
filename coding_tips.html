Using early returns

First tip is to use early return instead of using else condition. Most of the time the trick lies in using condition like !vehicles then return. If condition is false then will continue the code execution. Let's explain the below example. Function name as it suggest it will retrieve vehicles. The code simply does check if there is no vehicles in response then return as soon as possible. No need to check another condition as it will be already return. 


function getVehicles(response) {

	if(!response.vehicles) {
		return;
	}

	/**		more code 	**/
}

Using Ternary Operator

Ternary operators comes in handy when our code has only 1 line if conditions. I often use it for single line condition. Using it for nesting conditions get very complicated. It can also be used for returning values. Let's explain the code a little bit. As the function name suggest this function check if the weather is hot or not. We first initialize weather variable with winter and there may be more code in the between. At the end we return false if weather is winter or spring.

Syntax is as follow***(condition) ? 'condition is true' : 'Condition is false'

Compare the 2 versions and decide which one is more clean and concise. As I mentioned earlier I prefer ternary operator for single line condition


function isHot() {

	let weather = 'winter';

	/**		more code 	**/

	return weather == 'winter || spring' ? false : true
}

function isHot() {

	let weather = 'winter'

	/**		more code 	**/

	if(weather == 'winter' || weather == 'spring') {
		return true;
	}
	else {
		return false
	}
}

Passing no more than 3 parameters to a function

One of the best practices when working with function is to pass paramters. It is recommended that there should be no more than 3 parameters pass to a function(can pass more than that) for improve readibility. If want to improve code readibilty just pass no more than 3 parameters to a function else it will mess up.
Consider below 2 examples 


/** 1st approach **/

function buildHouse(bricks, cement, rooms, hasKitchen, hasWifi) {
	/** build a house **/
}

/** 2nd approach **/

function buildHouse(resources) {

	/**
		resources.bricks
		resources.cement
		resources.rooms
		resources.hasKitchen
		resources.hasWifi
	**/
}

Optional Chaining Operator(?.)

When accessing property of an object is null or undefined browsers crashes or stop further script execution. To avoid it. Use this operator. Other benefits includes it will not block further script execution.

function getRecords(response) {

	let records = null;

	if(response.status === 200) {
		response.records
	}
}

Toggle method for constraints

When changing classes based on condition it is better to use toggle method instead of using separate if else condition

function showTab(tabId) {
	document.querySelector(`#tab-{$tabId}`).classList.toggle('active', tabId)
}

Switch vs If Else

We may encounter some cases where there are limited and unique number of conditions. Using if else conditions make the code messy. Instead use switch statements for such scenarios.

function carPrice(price) {
	switch(price) {
		case 10: 
			alert("You can just buy a small toy car")
		break;

		case 50: 
			alert("You can just buy a medium toy car")
		break;

		case 100: 
			alert("You can just buy a big toy car")
		break;

		case 500: 
			alert("You can just buy a car for children to drive")
		break;

		case 10000: 
			alert("You can just buy a real car")
		break;

		default:
			alert("No car is available for such price")
	}
}

Using keyup event

Keyup event occurs as users types something in. This event is called every single character is pressed. Some use cases for it are:
1) Writing cover letter or application form used by Coursera.
2) Used by LinkedIn (sending personalize invites)
3) Writing tweets(used by twitter)

function getRecords(response) {
	let lengthElement = document.querySelector('span')
	let comment = document.querySelector('textarea')

	let commentCharacters = 100;
	let totalCharacters = 100;
	let length = 0;


	comment.addEventListener('keyup', (event) => {
			
		length = event.target.value.length
		
		commentCharacters = totalCharacters - length;

		if(commentCharacters >= 0) {
			document.querySelector('textarea + span').innerHTML = 
						commentCharacters + ' characters remaining';
		}

	})

}
